```json
      "searchMode": "search",

  }

```
- **Why QMD over builtin**: Haplo accumulates many sessions; QMD runs as a separate process so the growing index doesn't bloat the gateway's memory heap
- **Builtin problem**: sqlite-vec loads into the gateway process → same bloat pattern as before

### 6. Compaction Config Fix (Haplo)
- **Root cause of old bloat**: `compaction.mode: "safeguard"` only triggers at context overflow
- **MiniMax M2.5** was doing `memoryFlush` calls (not Gemini) → tool use failures → compaction never completing
- **Fix attempted**: Change to `mode: "auto"` + separate `compaction.model: "google/gemini-2.5-flash"`
- **Result**: OpenClaw 2026.3.2 only supports `"default"` and `"safeguard"` modes; `model` key not valid
- **Final state**: `mode: "default"` (OpenClaw's own default behavior)
- **Note for future**: When OpenClaw adds compaction model routing, set to Gemini for reliable tool use

## Key Lessons
1. **Rate Guard Ansible role patches npm files** — any npm reinstall of openclaw wipes the patches. Rate Guard must be re-applied after every openclaw upgrade.
2. **Safeguard compaction + bad tool use model = session bloat** — always use a model with reliable tool use for compaction (Gemini, Claude, GPT-4o).
3. **Orphan child processes don't die on gateway restart** — only a full `sudo reboot` clears them.
4. **QMD for high-session environments** — builtin SQLite loads into gateway process memory; QMD offloads to a subprocess.
5. **`openclaw gateway install` bug in 2026.3.2** — `systemctl --user is-enabled` fails before service file exists. Workaround: manually write service file.
