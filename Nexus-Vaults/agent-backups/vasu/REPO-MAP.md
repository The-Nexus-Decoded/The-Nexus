## Monorepo Rule — The-Nexus Only

All code, issues, and PRs go through The-Nexus monorepo.
Legacy standalone repos are deprecated.
Never create issues, branches, or PRs on standalone repos.

## Boundary Rules

- Unity and Unreal project code goes in `Arianus-Sky/projects/games/{project-name}/`
- Shared Unity tooling goes in `Arianus-Sky/shared/unity-tooling/`
- Shared Unreal tooling goes in `Arianus-Sky/shared/unreal-tooling/`
- Do NOT dump engine files at `Arianus-Sky/` root
- Game backend is primarily Haplo's domain. Coordinate closely, and do not take backend ownership unless explicitly assigned by Lord Xar.
- XR/spatial projects: coordinate with Samah; Unity/Unreal implementation is yours
- UI/UX surfaces on games: coordinate with Paithan for app-level UI, handle engine-side integration yourself

## Unity-Specific Gitignore Requirements

Always include these in `.gitignore` for Unity projects:
```
Library/
Temp/
Logs/
obj/
Build/
Builds/
UserSettings/
.vs/
*.csproj
*.sln
```

Never commit `Library/` — it is auto-generated by Unity and contains no source truth.

## Unreal-Specific Gitignore Requirements

Always include these in `.gitignore` for Unreal projects:
```
Binaries/
Build/
DerivedDataCache/
Intermediate/
Saved/
.vs/
*.VC.db
*.opensdf
*.opendb
*.sdf
*.sln
*.suo
*.xcodeproj
*.xcworkspace
```

Never commit `Intermediate/`, `Saved/`, or `DerivedDataCache/` — they are auto-generated and project-specific.

## New Project Creation Workflow

You are an **Architect Developer** — you propose and architect projects but do NOT create folders or tickets yourself.

1. Write a Project Spec using the template in REPO-MAP
2. Post spec in #coding and tag Zifnab
3. Zifnab creates the GitHub project structure and folder hierarchy
4. Wait for Zifnab's confirmation before writing any code
5. Architect-tier responsibility: the spec itself must include performance budget, target device floor, engine selection rationale, and a draw-call / memory budget before implementation begins

## Storage Protocol

Use `/data/` for:
- repositories (`/data/repos/The-Nexus/`)
- build outputs (`/data/logs/` or project dir)
- large assets not committed to git
- Unity `Library/` and Unreal `DerivedDataCache/` can be symlinked into `/data/engine-cache/{project-name}/` to keep the OS drive clean

Never use the OS drive for project data.
