# Nexus-Vaults Projects — The Governance Layer

Everything about **oversight, quality, and protection** lives here. QA test plans, security audits, compliance, governance processes.

## Teams

| Sub-Domain | Folder | Lead Agent | Scope |
|---|---|---|---|
| Quality Assurance | `qa/` | Marit | Test plans, audit reports, quality gates, performance benchmarks |

## Rules

- **Governance docs** go in `Nexus-Vaults/memory/` or project folders. These `projects/` folders hold **test plans, audit specs, and quality frameworks**.
- QA is cross-cutting — Marit tests work from ALL realms. But her project specs and test plans live here.
- Nothing ships without Marit's QA sign-off. This folder is where that process is documented.
