  <action>
Create two Ansible roles (base and tailscale) and the master playbooks.

**Role: tailscale** (`roles/tailscale/tasks/main.yml`):
1. Install Tailscale via official install script: `curl -fsSL https://tailscale.com/install.sh | sh`
2. Start Tailscale with auth key: `tailscale up --authkey={{ tailscale_auth_key }}`
3. Wait for Tailscale IP assignment: `tailscale ip -4` (register result)
4. Display the assigned Tailscale IP (debug msg for the user to note)
5. Handler: restart tailscale service

**Role: base** (`roles/base/tasks/main.yml`):
1. Update apt cache and install packages: `build-essential`, `curl`, `ufw`, `fail2ban`, `unattended-upgrades`
2. Create `openclaw` user (if not already the ansible_user) with sudo privileges
3. Copy SSH public key to `openclaw` user's authorized_keys (using `authorized_key` module)
4. Deploy SSH hardening config from template `99-hardening.conf.j2` to `/etc/ssh/sshd_config.d/99-hardening.conf`
5. Configure UFW (INFRA-05 per-server):
   - Default deny incoming, allow outgoing
   - Allow all traffic on `tailscale0` interface (`ufw allow in on tailscale0`)
   - Allow Tailscale control plane: `ufw allow 41641/udp`
   - Allow shared ports from `ufw_allowed_ports` (defined in `all.yml`)
   - Allow per-server ports from `ufw_extra_ports` (defined in per-server group_vars, empty by default -- provides the per-server override mechanism required by INFRA-05; will be populated in later phases when services are deployed)
   - Enable UFW
6. Configure Fail2ban from template `jail.local.j2`:
   - SSH jail enabled, maxretry from var, bantime from var
7. Enable unattended-upgrades for security patches
8. Handlers: restart ssh, restart fail2ban, reload ufw

**Template: 99-hardening.conf.j2:**
```
PasswordAuthentication no
PermitEmptyPasswords no
PermitRootLogin no
PubkeyAuthentication yes
MaxAuthTries 3
LoginGraceTime 30
ClientAliveInterval 300
ClientAliveCountMax 2
AllowUsers *@100.64.0.0/10
```

**Template: jail.local.j2:**
```
[sshd]
enabled = true
port = {{ ssh_port }}
maxretry = {{ fail2ban_maxretry }}
bantime = {{ fail2ban_bantime }}
```

**Playbook: site.yml** (master playbook):
```yaml
- name: Provision and harden all servers
  hosts: all
  become: true
  roles:
    - tailscale
    - base
```

**Playbook: harden.yml** (run independently to re-harden):
```yaml
- name: Harden SSH, UFW, Fail2ban
  hosts: all
  become: true
  roles:
    - base
```

**CRITICAL ORDER:** The `site.yml` playbook runs Tailscale FIRST (so the tailscale0 interface exists), THEN base (which configures UFW to allow tailscale0). If reversed, UFW blocks Tailscale setup.

**Anti-lockout safeguard:** The base role should include a pre-task that verifies `tailscale0` interface exists before applying UFW rules. If it does not exist, skip UFW and warn.
