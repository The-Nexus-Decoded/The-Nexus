<task type="checkpoint:decision" gate="blocking">
  <name>Task 1: Decide iMessage bridge availability</name>
  <decision>Is a Mac available to serve as the iMessage bridge for critical alerts?</decision>
  <context>
iMessage requires a macOS machine running Messages.app, reachable by all 3 Linux servers via Tailscale SSH. This Mac must be always-on (or at least on during trading hours). If no Mac is available, ntfy.sh (self-hosted push notifications) is the fallback for critical alerts -- it delivers push notifications to iOS/Android without Apple dependency.

This decision affects Task 2 configuration: if Mac available, configure the imsg-ssh bridge and NOTIF-02 is fulfilled. If not, configure ntfy.sh as a temporary critical alert channel -- but note that choosing ntfy.sh **defers** NOTIF-02 (iMessage requirement) to a future phase when a Mac becomes available.
  </context>
  <options>
    <option id="mac-available">
      <name>Mac is available for iMessage bridge</name>
      <pros>Native iMessage to owner's phone, no additional services to run, integrates with OpenClaw's built-in iMessage channel</pros>
      <cons>Mac must stay on and connected to Tailscale, requires Full Disk Access and Remote Login setup, imsg is "legacy" (BlueBubbles recommended for new setups)</cons>
    </option>
    <option id="no-mac-ntfy">
      <name>No Mac available -- use ntfy.sh as temporary fallback (DEFERS NOTIF-02)</name>
      <pros>No Apple hardware dependency, self-hosted option available, works on iOS and Android, simple HTTP POST API, can be added to any server without special bridging</pros>
      <cons>Does NOT fulfill NOTIF-02 (iMessage requirement) -- NOTIF-02 will remain open until a Mac is available. Not iMessage (separate app for notifications), requires running ntfy server or using ntfy.sh cloud, less integrated with OpenClaw (custom script vs built-in channel)</cons>
    </option>
  </options>

**Step 3: Create notifications role.**

`roles/notifications/tasks/main.yml`:

1. **Look up server_user UID (required for user-scope systemd restart):**
   ```yaml
   - name: Look up UID for {{ server_user }}
     command: id -u {{ server_user }}
     register: server_user_id_result
     changed_when: false

   - name: Set server_user_uid fact
     set_fact:
       server_user_uid: "{{ server_user_id_result.stdout }}"
   ```

2. **Deploy iMessage SSH wrapper (if mac-available):**
   ```yaml
   - name: Deploy imsg-ssh wrapper script
     template:
       src: imsg-ssh.j2
       dest: /home/{{ server_user }}/.openclaw/scripts/imsg-ssh
       owner: "{{ server_user }}"
       mode: '0755'
     when: imsg_enabled | default(false)
   ```

3. **Generate SSH key for Mac bridge (if mac-available):**
   ```yaml
   - name: Generate SSH key for Mac bridge
     user:
       name: "{{ server_user }}"
       generate_ssh_key: yes
       ssh_key_type: ed25519
       ssh_key_file: /home/{{ server_user }}/.ssh/id_mac_bridge
     when: imsg_enabled | default(false)

   - name: Display public key for Mac authorized_keys
     command: cat /home/{{ server_user }}/.ssh/id_mac_bridge.pub
     register: mac_bridge_pubkey
     when: imsg_enabled | default(false)

   - name: Print Mac bridge public key (user must add to Mac)
     debug:
       msg: "Add this key to {{ imsg_mac_user }}@{{ imsg_mac_tailscale_ip }}:~/.ssh/authorized_keys: {{ mac_bridge_pubkey.stdout }}"
     when: imsg_enabled | default(false)
   ```

4. **Deploy ntfy notify script (if no-mac-ntfy):**
   ```yaml
   - name: Deploy ntfy critical alert script
     template:
       src: notify-critical.sh.j2
       dest: /home/{{ server_user }}/.openclaw/scripts/notify-critical.sh
       owner: "{{ server_user }}"
       mode: '0755'
     when: not (imsg_enabled | default(false))
   ```

5. **Restart OpenClaw gateway to pick up channel config:**
   ```yaml
   - name: Restart OpenClaw gateway for channel config
     systemd:
       name: openclaw-gateway
       state: restarted
       scope: user
     become_user: "{{ server_user }}"
     environment:
       XDG_RUNTIME_DIR: "/run/user/{{ server_user_uid }}"
   ```

6. **Wait for gateway to come up:**
   ```yaml
   - name: Wait for gateway health check
     uri:
       url: http://127.0.0.1:18789/health
       return_content: yes
     register: health
     retries: 10
     delay: 5
     until: health.status == 200
   ```

**Template: imsg-ssh.j2:**

```bash
#!/usr/bin/env bash
exec ssh -T -i /home/{{ server_user }}/.ssh/id_mac_bridge {{ imsg_mac_user }}@{{ imsg_mac_tailscale_ip }} /usr/local/bin/imsg "$@"
```
