# Pryan-Fire Projects — The Execution Layer

Everything that gets **built and run** lives here. Backend services, trading algorithms, API engines, data processing.

## Teams

| Sub-Domain | Folder | Lead Agent | Scope |
|---|---|---|---|
| Backend Services | `backend/` | Haplo | APIs, backend logic, CI/CD tooling, process supervisors |
| Trading & Execution | `trading/` | Hugh | Trading algorithms, financial connectors, risk management |

## Rules

- **Code** goes in `Pryan-Fire/haplos-workshop/`, `hughs-forge/`, or `zifnabs-scriptorium/`. These `projects/` folders hold **specs, plans, and status docs** — not source code.
- If a project is primarily a backend service that powers a frontend, it lives here. The frontend spec lives in Arianus-Sky.
- Trading strategies and financial logic are Hugh's exclusive domain. Haplo builds infrastructure; Hugh builds trading logic.
