# Abarrach-Stone Projects — The Data Layer

Everything about **persistence, data, and knowledge** lives here. Schemas, analytics, data pipelines, historical records.

## Teams

| Sub-Domain | Folder | Lead Agent | Scope |
|---|---|---|---|
| Analytics & Data | `analytics/` | (cross-team) | Data models, reporting specs, analytics pipelines, business intelligence schemas |

## Rules

- **Code** goes in `Abarrach-Stone/schemas/`, `pipelines/`, or `archives/`. These `projects/` folders hold **specs and plans** — not source code.
- Any project that is primarily about data modeling, storage design, or analytics reporting belongs here.
- Market data schemas from trading (Hugh) and business intelligence data from sales (Sang-drax) both route here for the data layer.
