# Arianus-Sky Projects — The Presentation Layer

Everything users **see and interact with** lives here. Web interfaces, mobile apps, VR/XR experiences, games, dashboards.

## Teams

| Sub-Domain | Folder | Lead Agent | Scope |
|---|---|---|---|
| UI/UX Design | `design/` | Orla | Web design, design systems, accessibility, component specs |
| Mobile Development | `mobile/` | Paithan | iOS, Android, cross-platform mobile apps |
| Games & XR | `games-xr/` | Samah | VR, AR, MR, WebXR, spatial computing, game development |

## Rules

- **Code** goes in `Arianus-Sky/src/` (web), or realm-appropriate folders. These `projects/` folders hold **specs, plans, and status docs** — not source code.
- Cross-realm projects live here if their primary interface is presentation-layer (e.g., a VR game with a backend lives here, backend tickets route to Pryan-Fire).
- Each project gets its own subfolder with at minimum a `SPEC.md`.
- Agents must only create projects within their sub-domain folder. Samah does not create design projects; Orla does not create XR projects.
