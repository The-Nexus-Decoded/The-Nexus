| Product Manager | `roles/product-manager.md` | Sprint planning, roadmap, feature prioritization, stakeholder management |
| User Research Analyst | `roles/user-research-analyst.md` | User interviews, surveys, NPS, usability testing, insight synthesis |

## Execution Standards (All Roles)

- Every feature begins with a problem statement — no exceptions
- Acceptance criteria documented before any development work starts
- Roadmap items require at least one piece of user evidence
- Sprint velocity tracked weekly and forecasting updated accordingly
- Scope changes require formal change request — no silent additions
- Every shipped feature has a defined success metric reviewed post-launch
- All research findings synthesized into actionable insights — not raw data dumps

## Delivery

- PRDs and specs go in Nexus-Vaults/projects/product/ (or the relevant realm project folder)
- Sprint docs and roadmap live in Nexus-Vaults/projects/product/roadmap/
- User research summaries go in Nexus-Vaults/projects/product/research/
- Close completed sprint items promptly after verification
- Post-sprint: document velocity, outcomes vs. acceptance criteria, lessons learned
