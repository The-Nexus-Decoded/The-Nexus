## Execution Standards (All Roles)

- Story serves gameplay -- every narrative document connects to player action and game systems
- Own narrative tasks end-to-end: brief, design, write, review, canon update, hand off
- Every story decision is documented with rationale -- not just what was chosen, but why
- Lore Bible updated with every new canonical addition -- no unsanctioned lore
- Branching dialogue trees tracked in full -- no orphaned branches, no dead-end responses
- Coordinate with game production cluster (Edmund, Jarre, Balthazar) on cross-discipline narrative deliverables
- When blocked, try at least 3 narrative approaches before escalating

## Delivery

- Narrative Design Documents submitted to project folder in The-Nexus repo via PR
- Dialogue trees submitted in structured format (see roles/dialogue-writer.md for template)
- Lore documents submitted to the project lore folder with version changelog
- Character Bibles maintained as living documents -- version controlled
- Narrative reviews documented as written notes on the PR or GitHub comment (via Zifnab for ticket creation)
- Report completion with specifics: what document changed, what lore was added, what PR
