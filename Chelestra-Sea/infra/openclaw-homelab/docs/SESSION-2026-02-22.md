## Gotchas & Lessons Learned

### OpenClaw Auth
- **`openclaw models auth paste-token`** launches a TUI that doesn't work well with piped input — key appears character-by-character but may not save. Writing directly to `auth-profiles.json` is more reliable.
- **Auth-profiles.json location**: `/home/openclaw/.openclaw/agents/main/agent/auth-profiles.json`
- **Env var for Gemini must be `GEMINI_API_KEY`** — NOT `GOOGLE_API_KEY`. OpenClaw looks for `GEMINI_API_KEY` specifically. Setting both causes a harmless "Both are set, using GOOGLE_API_KEY" warning but OpenClaw only reads `GEMINI_API_KEY`.
- **`openclaw models status`** shows "Missing auth" when run from CLI without env vars, even if the gateway process has them via systemd override. This is cosmetic — the gateway itself works fine.

### Google Gemini API
- **New projects get `limit: 0` on free tier** until billing is linked to the Cloud project.
- **Adding a credit card isn't enough** — you must also enable the **Generative Language API** in Google Cloud Console.
- **Keys created BEFORE billing was linked are stuck on zero-quota bucket**. Delete the old key and create a new one AFTER billing is active.
- **Tier 1** (after billing) gives you the free usage allowance: ~100 req/day for Pro, ~250 req/day for Flash.
- **The upgrade path**: AI Studio → create key → Google Cloud Console → enable billing → enable Generative Language API → delete old key → create new key.

### OpenAI API
- **ChatGPT Pro subscription ≠ API access**. They're separate products.
- **Codex (5 hours/week in ChatGPT Pro)** is a web UI feature, not available via API.
- **API requires prepaid credits** at platform.openai.com/settings/organization/billing.
- **Service account keys** work but need credits loaded.
- **GPT-4.1 is outdated for coding** — superseded by Codex models (codex-mini-latest at $1.50/$6.00, GPT-5 Codex, etc.)

### Ollama / Local Models
- **Qwen 2.5 Coder 7B** scores 88.4% on HumanEval (good!) but can't handle OpenClaw's system prompt at default 4096 context window.
- **OLLAMA_NUM_CTX** must be at least 16384 for OpenClaw. Set in systemd override at `/etc/systemd/system/ollama.service.d/override.conf`.
- Even at 16384, Qwen 7B produces empty or tool-call-only responses for complex tasks — too small for OpenClaw's full agent loop.

### Discord Bot
- **"channels unresolved"** in logs is cosmetic — bot still works.
- **"no-mention" skip** means `requireMention` is true and the message didn't @mention the bot.
- **Bot shows "typing" but sends nothing** = model returned empty payload (usually context window too small).
- **DNS failures (`getaddrinfo EAI_AGAIN`)** kill the Discord connection. Fix DNS first.
- **Message Content Intent** must be enabled in Discord Developer Portal → Bot settings.

### Gateway Systemd Override
- **Location**: `~/.config/systemd/user/openclaw-gateway.service.d/ollama.conf`
- **Current contents**:
  ```ini
  [Service]
  Environment="OLLAMA_API_KEY=ollama-local"
  Environment="OPENAI_API_KEY=<your-openai-key>"
  Environment="GEMINI_API_KEY=<your-gemini-key>"
  ```
- After any change: `systemctl --user daemon-reload && openclaw gateway restart`

### Moving the Server
- **Tailscale** reconnects automatically — same 100.x.x.x IP regardless of physical location.
- **MAC-based NIC matching** in Netplan means the network comes up at any location with DHCP.
- **Agent memory/sessions** persist on disk under `~/.openclaw/` and `/data/openclaw/`.
- **Discord bot** reconnects automatically.
- **Only LAN IP changes** — doesn't matter since everything uses Tailscale.
