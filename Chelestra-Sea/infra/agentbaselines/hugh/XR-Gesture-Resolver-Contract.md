# XR Gesture Resolver Contract

**Version:** 1.0-draft  
**Status:** Pending Review  
**Owners:** Paithan (Mobile), Orla (UX), Samah (Spatial)

---

## 1. Overview

This contract defines the interface between the Mobile App Layer and the Spatial Layer for gesture-based interaction in the XR experience. It enables bidirectional intent resolution with stateful handoff and coordinated presentation.

---

## 2. Communication Pattern

```
Mobile App                          Spatial Layer
     │                                    │
     ├──[GestureIntent + Context]──────► │
     │                                    │ (resolve intent)
     │◄───[state update/confirmation]─────┤
     │                                    │
     │  [cache state for return]          │
     │  (keyed by returnToken)           │
```

### 2.1 Principles

1. **Bidirectional** — Mobile↔Spatial both emit `GestureIntent`. Same contract both directions.
2. **Stateful handoff** — Mobile persists context cache, keyed by `returnToken`. Restores on return.
3. **Spatial coords** — Mobile normalizes to spatial 3D space, not screen 2D.
4. **Resolve layer** — Spatial interprets intent. Mobile just emits.

## 3. GestureIntent Contract

### 3.1 Payload

```swift
struct GestureIntent {
    let action: GestureAction        // move | rotate | scale | cast | charge | grab
    let source: GestureSource         // gesture | menu | haptic
    let target: String?               // entity_id, null if none
    let parameters: [String: Any]?     // action-specific params
    let confidence: Double            // 0.0-1.0
    let transitionPoint: String      // where in flow we are
    let returnToken: UUID             // for stateful return
    let timestamp: UInt64             // unix_ms
}
```

### 3.2 Enums

```swift
enum GestureAction: String {
    case move, rotate, scale, cast, charge, grab
}

enum GestureSource: String {
    case gesture, menu, haptic
}
```

### 3.3 Rules

- **Confidence threshold:** 0.85 minimum for override. Below = reject/ fallback.
- **Return token:** Required. Generated by Mobile. Used to restore context on return.
- **Transition point:** Flow state identifier (e.g., `selecting`, `manipulating`, `casting`).

## 4. Intent Resolution Response

### 4.1 Success

```swift
struct IntentResolution {
    let status: "resolved"
    let returnToken: UUID
    let selectedObject: String?        // entity_id
    let actionApplied: GestureAction
    let timestamp: UInt64
    let ttl_ms: Int                  // intentional window for validity
}
```

### 4.2 Error

```swift
struct IntentError {
    let status: "error"
    let returnToken: UUID
    let errorCode: ErrorCode
    let message: String
    let fallbackSuggested: Bool
}

enum ErrorCode: String {
    case object_not_found
    case out_of_bounds
    case confidence_too_low
    case timeout
    case invalid_transition
}
```

## 5. State Broadcast

Emitted by Spatial Layer to keep Mobile in sync.

```swift
struct SpatialState {
    let selectedObject: String?       // entity_id, null if none
    let cameraPose: CameraPose
    let timestamp: UInt64
    let ttl_ms: Int                  // validity window
}

struct CameraPose {
    let position: vec3
    let rotation: quat               // quaternion
}
```

**Broadcast triggers:**
- Object selection changed
- Camera moved significantly (>5cm or >5°)
- Every 500ms heartbeat if idle

## 7. Queue & Preview Management

### 7.1 Preview Queue

```swift
struct QueueConfig {
    let preview_queue_max: Int       // default: 3
    let queue_strategy: QueueStrategy
    let on_timeout: TimeoutAction
}

enum QueueStrategy: String {
    case local_then_push   // queue locally, push when ready
    case sync_on_foreground  // sync when app returns to foreground
    case drop              // drop extras
}

enum TimeoutAction: String {
    case queue   // keep in queue
    case drop    // discard
    case retry   // re-attempt
}
```

### 7.2 Timeout

- **Default timeout:** 100ms for intent→resolution round-trip
- **Retry logic:** 1 retry on timeout, then `on_timeout` applies

## 8. Continuous Actions (cast, charge)

For `cast` and `charge`, emit **start/stop events**, not continuous updates:

```swift
// Start
{ "action": "cast", "transitionPoint": "cast_start", ... }

// Progress (optional, if visual feedback needed)
// May emit every 100ms during charge

// Stop
{ "action": "cast", "transitionPoint": "cast_complete", ... }
```

**Latency budget:** 100ms max for intent→confirm round-trip to maintain spatial coherence.

## 10. Entity Targeting (The "What")

The Spatial Layer exposes methods to resolve 2D/3D inputs to entity selection.

### 10.1 hitTest — Mobile Touch → VR Selection

```swift
func hitTest(screenPoint: vec2) -> HitTestResult
```

**Input:** Normalized screen coordinates (0-1, origin bottom-left)  
**Output:**

```swift
struct HitTestResult {
    let entityId: String?           // null if miss
    let worldPosition: vec3?         // hit point in world space
    let normal: vec3?               // surface normal at hit
    let distance: Float?            // distance from camera
}
```

**Use case:** Mobile tap/hold → select entity in VR view.

---

### 10.2 raycast — VR Gesture → Entity Mapping

```swift
func raycast(origin: vec3, direction: vec3) -> RaycastResult
```

**Input:** World-space ray (typically from hand controller)  
**Output:**

```swift
struct RaycastResult {
    let entityId: String?            // null if miss
    let hitPosition: vec3?           // exact hit point
    let normal: vec3?                // surface normal
    let distance: Float              // ray distance to hit
}
```

**Use case:** VR pointing gesture → entity selection. Direction is normalized.

---

### 10.3 getEntityState — Mobile Preview Sync

```swift
func getEntityState(entityId: String) -> EntityState
```

**Output:**

```swift
struct EntityState {
    let entityId: String
    let position: vec3
    let rotation: quat
    let scale: vec3
    let locked: Bool                 // true = being manipulated elsewhere
    let lastModified: UInt64         // unix_ms
}
```

**Use case:** Mobile preview sync — fetch current transform + lock state before rendering preview.

## 11. Error Handling Summary

| Scenario | Response | Retry? |
|----------|----------|--------|
| Object not found | `error: object_not_found` | No — user must re-select |
| Out of bounds | `error: out_of_bounds` | No — clamp or reject |
| Confidence < 0.85 | `error: confidence_too_low` | Yes — re-acquire |
| Timeout (>100ms) | `on_timeout` applies | Configurable |
| Invalid transition | `error: invalid_transition` | No |
