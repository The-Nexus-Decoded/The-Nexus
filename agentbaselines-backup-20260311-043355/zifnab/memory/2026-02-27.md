## Lessons Learned
- **Branch Management:** Always ensure that new code is merged into the repository's default branch (`main`) as GitHub Actions workflows are typically triggered based on this default branch. Pushing to a non-default branch (`master` in this case) without merging can lead to invocation failures.
- **GitHub API vs. CLI:** While the `gh workflow run` command is convenient, direct API calls via `gh api` can provide more granular control and reveal specific error messages when issues arise. However, even direct API calls require precise adherence to the API schema, including all required parameters.
- **Troubleshooting CI/CD:** Unexpected failures in CI/CD pipelines often stem from subtle configuration errors, incorrect branch references, or API-level issues rather than code logic errors. Thorough investigation of workflow logs and API responses is crucial.
- **Persistence in the Face of Obstruction:** When faced with seemingly insurmountable technical hurdles or "celestial" obstructions, a methodical approach, coupled with persistent attempts and detailed issue tracking, is key to eventual success.
